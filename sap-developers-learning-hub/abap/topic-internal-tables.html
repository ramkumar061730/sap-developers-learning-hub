Internal Tables & Programming

Standard, Sorted, and Hashed tables — operations, performance tips, FAE rules, and interview Q&A.

What is an Internal Table?

An Internal Table is a temporary in-memory table (collection of records) that exists only during program execution. It is program-scoped and dynamic in size. Records are accessed one-by-one.

📌 Internal tables are destroyed when the program ends (unlike database tables).

Types of Internal Tables

💡 Rule of thumb: use Hashed for frequent key lookups, Sorted for range + binary search, Standard for simple sequential processing.

Operations on Internal Tables

APPEND — Add to end

APPEND <wa> TO <it>. APPEND LINES OF <it1> TO <it2>.

INSERT — Add at specific position

INSERT <wa> INTO <it> INDEX <n>.

READ — Retrieve a record

READ TABLE <it> INTO <wa> INDEX <n>. READ TABLE <it> INTO <wa> WITH KEY <field> = <value>.

MODIFY — Update records

MODIFY <it> FROM <wa> TRANSPORTING <field> WHERE <condition>.

DELETE — Remove records

DELETE <it> WHERE <condition>. DELETE ADJACENT DUPLICATES FROM <it> COMPARING <field>.

LOOP — Iterate records

LOOP AT <it> INTO <wa> [WHERE <condition>]. ENDLOOP.

SORT — Sort records

SORT <it> BY <field> ASCENDING. SORT <it> BY <field> DESCENDING.

DESCRIBE TABLE — Count records

DESCRIBE TABLE <it> LINES <lv_lines>.

💡 After READ TABLE, always check SY-SUBRC (0 = found) and SY-TABIX for index.

CLEAR vs REFRESH vs FREE

  • CLEAR <WA>: resets work area to initial values (blank/zero). Does not free memory.
  • REFRESH <IT>: deletes all rows but retains memory allocation.
  • FREE <IT>: deletes all rows and releases allocated memory back to system.

📌 Use FREE when you are done with a large table and want to reduce memory usage.

FOR ALL ENTRIES (FAE)

FAE fetches data from a DB table for each entry in an internal table — like a mass select using internal table values as filter.

Rules / Restrictions

  • FAE internal table must NOT be empty (check IS INITIAL)
  • Duplicate records in result are removed automatically
  • Cannot use ORDER BY, GROUP BY, HAVING with FAE
  • Must have common fields between driver itab and DB table (in WHERE)

⚠️ If your driver itab is empty and you still run FAE, you may accidentally select the whole DB table. Always guard with IF itab IS NOT INITIAL.

Interview Q&As

QWhat is an Internal Table in SAP ABAP? How is it different from a database table?
A
Internal table is a temporary in-memory structure that exists only during program execution. Database table is permanently stored in the database. Key differences: internal table is program-scoped, dynamically sized, no commit needed; DB table is persistent and requires DB operations.
QWhat are the three types of internal tables? Explain each with their search method.
A
  • Standard: accepts duplicates; linear search by default (binary optional).
  • Sorted: auto-sorted by key; binary search is mandatory; unique/non-unique key possible.
  • Hashed: unique key only; O(1) access using hash; no index.
QDifference between APPEND, INSERT, and COLLECT in internal tables?
A
APPEND: adds record to the end (standard tables).
INSERT: adds record at a specific index / maintains sort in sorted tables.
COLLECT: sums numeric fields for same key instead of adding a new row.
QWhat is the difference between CLEAR, REFRESH, and FREE?
A
CLEAR resets work area values; CLEAR of table with header line clears only header line. REFRESH deletes rows but keeps memory allocation. FREE deletes rows and releases memory to the system.
QWhat is FOR ALL ENTRIES? What are its restrictions?
A
FAE executes a SELECT for each row in an internal table (join-like behavior in ABAP layer). Restrictions: driver itab must not be empty; duplicates removed; no ORDER BY/GROUP BY/HAVING/aggregates; no UPTO N ROWS; must include at least one driver field in WHERE.
QDifference between SELECT SINGLE and UPTO 1 ROWS?
A
SELECT SINGLE needs full primary key for best performance and returns exactly one row. UPTO 1 ROWS is more flexible on WHERE fields and returns the first match but is slightly less optimal.
QWhat is a Header Line in an internal table?
A
Header line is an implicit work area with same name as the internal table (older ABAP using WITH HEADER LINE). Modern ABAP prefers explicit separate work areas for clarity.
QHow do you delete adjacent duplicates from an internal table?
A
First SORT the internal table, then:
DELETE ADJACENT DUPLICATES FROM itab COMPARING field1 field2.
Sorting is mandatory so duplicates become adjacent.
QWhat is DESCRIBE TABLE used for?
A
DESCRIBE TABLE itab LINES lv_lines gives the number of rows. Used before processing and before using FAE.
QHow do you read a specific record from an internal table?
A
By index: READ TABLE itab INTO wa INDEX n.
By key: READ TABLE itab INTO wa WITH KEY field = value.
For sorted/hashed tables: use WITH TABLE KEY for optimized search. Always check SY-SUBRC.
QDifference between MODIFY single record and multiple records?
A
Single: MODIFY itab FROM wa INDEX n.
Multiple: MODIFY itab FROM wa TRANSPORTING field WHERE condition.
TRANSPORTING updates only selected fields (preferred).
QWhat is the scope of an internal table?
A
Declared in main program (DATA) → global scope within program and accessible across events. Declared inside FORM/METHOD → local scope. Internal tables are not accessible outside the program and are destroyed after execution ends.
