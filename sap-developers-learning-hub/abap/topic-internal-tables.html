Internal Tables & Programming

Standard, Sorted, Hashed tables — operations, memory handling, and common interview questions.

What is an Internal Table?

An Internal Table is a temporary in-memory table (collection of records) that exists only during program execution. Access is done record-by-record.

📌 Internal tables are program-scoped and are destroyed when the program ends.

Operations on Internal Tables

APPEND — Add to end

APPEND <wa> TO <it>. APPEND LINES OF <it1> TO <it2>.

INSERT — Add at position

INSERT <wa> INTO <it> INDEX <n>.

READ — Retrieve a record

READ TABLE <it> INTO <wa> INDEX <n>. READ TABLE <it> INTO <wa> WITH KEY <field> = <value>.

MODIFY — Update records

MODIFY <it> FROM <wa> TRANSPORTING <field> WHERE <condition>.

DELETE — Remove records

DELETE <it> WHERE <condition>. DELETE ADJACENT DUPLICATES FROM <it> COMPARING <field>.

LOOP — Iterate records

LOOP AT <it> INTO <wa> [WHERE <condition>]. ENDLOOP.

SORT — Sort records

SORT <it> BY <field> ASCENDING. SORT <it> BY <field> DESCENDING.

DESCRIBE TABLE — Count rows

DESCRIBE TABLE <it> LINES <lv_lines>.

CLEAR vs REFRESH vs FREE

  • CLEAR <wa|it>: clears content; does not free memory
  • REFRESH <it>: clears all rows; does not free memory allocation
  • FREE <it>: clears rows and releases memory back to system

💡 Use FREE when you’re done with a large internal table to reduce memory footprint.

FOR ALL ENTRIES (FAE)

FAE fetches data from a database table for each entry in an internal table (mass SELECT behavior).

Rules / Restrictions

  • FAE internal table must not be empty (check IS INITIAL)
  • Duplicate result rows are removed automatically
  • Cannot use ORDER BY, GROUP BY, HAVING with FAE
  • Must have common fields between internal table and DB table in WHERE

⚠️ Always guard FAE with: IF itab IS NOT INITIAL. Otherwise it may return the entire DB table.

Interview Q&As

QWhat is an Internal Table? How is it different from a DB table?
AAn internal table is temporary, in-memory, program-scoped and exists only during execution. A DB table is persistent, stored in the database, accessible system-wide, and needs DB operations for reads/writes.
QExplain Standard vs Sorted vs Hashed tables (search method).
AStandard: linear search (binary optional). Sorted: automatically sorted by key, binary search. Hashed: unique key, O(1) key access via hash algorithm, no index.
QDifference between CLEAR, REFRESH, and FREE?
ACLEAR resets values but doesn’t free memory. REFRESH deletes rows but keeps memory allocation. FREE deletes rows and releases memory back to the system.
QWhat is FOR ALL ENTRIES? Key restrictions?
AFAE selects DB data based on values in an internal table. The driver table must not be empty; duplicates are removed; ORDER BY/GROUP BY/HAVING aren’t allowed; common fields must exist in WHERE.
QHow do you delete adjacent duplicates?
ASORT the table first, then use: DELETE ADJACENT DUPLICATES FROM itab COMPARING field.
