Topic 02

Report Programming

Everything you need for ABAP reporting: classical events, interactive drill-down, ALV basics, and interview-ready Q&A.

Overview (RICEF/W)

RICEF/W stands for Reports, Interfaces, Conversions, Enhancements, Forms, Work Flow. These are the key development object types in SAP ABAP.

📌 Reports are used to fetch, process, and present business data (lists/ALV/exports) based on selection-screen inputs.

Types of Reports

  • Classical Reports — flat list output (WRITE-based), minimal interaction.
  • Interactive Reports — drill-down output using list events (double-click etc.).
  • ALV Reports — standard table display with sorting/filtering/totals/export.

Classical Report Events

Classical reports follow an event-driven sequence. Main data retrieval and output logic is typically placed in START-OF-SELECTION.

EventDescription
LOAD-OF-PROGRAMLoads the program into memory for execution.
INITIALIZATIONInitialize default values of variables and selection-screen fields.
AT SELECTION-SCREEN OUTPUTDynamically modify selection screen (hide/disable fields).
AT SELECTION-SCREENValidate multiple fields together (cross-field validation).
AT SELECTION-SCREEN ON <field>Validate a single input field.
AT SELECTION-SCREEN ON VALUE-REQUESTProvide search help (F4) for an input field.
AT SELECTION-SCREEN ON HELP-REQUESTProvide help information (F1) for an input field.
START-OF-SELECTIONMain processing — fetch data and build output.
END-OF-SELECTIONEnd marker after data processing.
TOP-OF-PAGEHeader printed on top of each page.
END-OF-PAGEFooter printed at bottom of each page.

💡 If you don’t write START-OF-SELECTION explicitly, ABAP places your top-level statements there by default.

Interactive Reports

Interactive reporting allows users to drill down to details. When a user interacts (double-click / toolbar), ABAP triggers list events and you can create a secondary list.

EventPurpose
AT LINE-SELECTIONTriggered on double-click in the output list.
AT USER-COMMANDHandle custom toolbar/menu actions via function codes.
TOP-OF-PAGE DURING LINE-SELECTIONHeader for the secondary list pages.
AT PF<n>Handle function key presses (F1–F12).
SET PF-STATUSAttach a custom GUI status to the report.

📌 Tip: Use HIDE in the primary list to store keys and read them again during AT LINE-SELECTION.

ALV Reports (ABAP List Viewer)

ALV is the standard SAP display framework for tables. It provides sorting, filtering, totals, layout variants, column selection, and export (Excel, etc.).

Types of ALV

  • Simple ALV — single list/grid display
  • Blocked ALV — multiple lists displayed consecutively
  • ALV Hierarchical Sequential — hierarchical (parent-child) display

Common ALV Function Modules

Function ModuleUse
REUSE_ALV_GRID_DISPLAYDisplay result in a grid
REUSE_ALV_LIST_DISPLAYDisplay result in a list
REUSE_ALV_FIELDCATALOG_MERGEBuild / merge field catalog

💡 ALV requires a field catalog (IT_FIELDCAT) to describe column properties (text, output length, key fields, etc.).

Control Level Processing (AT Events)

Control level events are processed within a LOOP on a (usually sorted) internal table to handle group headers/footers/subtotals.

EventTriggered WhenTypical Use
AT FIRSTFirst row is processedPrint report header / init totals
AT NEW <field>Group field value changesPrint group header
AT END OF <field>End of a groupPrint subtotal / group footer
AT LASTLast row is processedPrint grand totals / end summary

⚠️ These events work correctly when data is sorted by the control field(s).

Interview Q&As

QWhat is a Classical Report in ABAP? How is it different from Interactive Report?
A
Classical Report: simple list-based report that displays data in a flat list format; user cannot interact with output.

Interactive Report: extends classical report by allowing user interaction — double-clicking a line on the primary list generates a secondary (detail) list. Uses events like AT LINE-SELECTION and AT USER-COMMAND.
QWhat are the events in a Classical Report? Explain each event.
A
LOAD-OF-PROGRAM: loads program into memory.
INITIALIZATION: default values before selection screen.
AT SELECTION-SCREEN OUTPUT: dynamic selection-screen modification.
AT SELECTION-SCREEN ON <field>: validate a single field.
AT SELECTION-SCREEN: validate multiple fields together.
AT SELECTION-SCREEN ON VALUE-REQUEST: F4 help.
START-OF-SELECTION: main processing (fetch/process/write).
END-OF-SELECTION: end marker after main processing.
TOP-OF-PAGE: page header.
END-OF-PAGE: page footer.
QWhat is the difference between AT SELECTION-SCREEN and AT SELECTION-SCREEN ON <field>?
A
AT SELECTION-SCREEN is triggered after the full selection-screen input is collected and is used for cross-field validation.

AT SELECTION-SCREEN ON <field> is triggered when the user leaves a specific field and is used for single-field validation.
QWhat is an Interactive Report? What events does it have?
A
An Interactive Report allows users to drill down by interacting with the output list.

Events include:
  • AT LINE-SELECTION (double-click)
  • AT USER-COMMAND (custom toolbar action)
  • TOP-OF-PAGE DURING LINE-SELECTION (secondary list header)
  • AT PF<n> (function key press)
Primary list is created in START-OF-SELECTION; secondary lists are created in interactive events.
QWhat is an ALV Report? What are its types?
A
ALV (ABAP List Viewer) is a standard framework to display tabular data with sorting, filtering, totals, column reordering, and export.

Types:
  • Simple ALV — single grid/list
  • Blocked ALV — multiple lists displayed together
  • Hierarchical Sequential ALV — tree-structured parent-child display
QWhat function modules are used in ALV reports?
A
Common ALV FMs:
  • REUSE_ALV_GRID_DISPLAY — grid display
  • REUSE_ALV_LIST_DISPLAY — list display
  • REUSE_ALV_FIELDCATALOG_MERGE — build field catalog
  • REUSE_ALV_COMMENTARY_WRITE — add commentary above/below ALV
QWhat is the difference between Parameter and Select-Options?
A
Parameter: single value only; typically used to fetch exact record.
Select-Options: supports single, multiple, and ranges; can be left blank to retrieve all data; creates a range table (SIGN/OPTION/LOW/HIGH).
QWhat are Control Level Events (AT Events)? Explain AT FIRST, AT NEW, AT LAST, AT END OF.
A
Control level events process grouped data within a LOOP on a sorted internal table.
  • AT FIRST: runs only for first record
  • AT NEW <field>: runs when value of the field changes
  • AT END OF <field>: runs at end of a group for that field
  • AT LAST: runs only for last record
Used for group headers, subtotals, and group footers.
QWhat is SY-SUBRC? What value does it hold on success?
A
SY-SUBRC is the return code of the last ABAP statement. 0 indicates success. Non-zero means not successful (e.g., not found, failed check, etc.).
QWhat are Variants in SAP ABAP?
A
Variants save selection-screen inputs (values, ranges, checkboxes) so they can be reused. Very important for background/scheduled jobs where manual input is not possible.
QWhat are the data types in SAP ABAP?
A
Numeric: I (integer), F (float), P (packed decimal).
Character: C, N, D (date), T (time), X (hex).
String: STRING, XSTRING.
QWhat is the difference between Report and Dynpro?
A
Report: generates a flat list output (WRITE), limited interaction.
Dynpro: interactive screen programming (input fields, buttons, menus) with data flow between screen and program.
QWhat is START-OF-SELECTION event?
A
START-OF-SELECTION is the main processing event after selection-screen events are complete. This is where the program fetches data (SELECT), processes it, and writes output. If not coded, ABAP assigns standalone statements to this event by default.
QWhat is the use of AT LINE-SELECTION event?
A
Triggered when the user double-clicks a line in the primary list. Commonly used to fetch additional details and display a secondary list. Useful system fields: SY-LILLI (selected line number), SY-LISEL (selected line content).
