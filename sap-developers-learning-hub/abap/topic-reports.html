Topic 02

Report Programming

Classical reports, interactive reports, and ALV — key events, patterns, and interview-ready explanations.

Overview (RICEF/W)

RICEF/W stands for Reports, Interfaces, Conversions, Enhancements, Forms, Work Flow. These are the key development object types in SAP ABAP.

📌 In ABAP projects, “Reports” are one of the most common deliverables—used for data extraction, analysis, and business outputs.

Types of Reports

  • Classical Reports
  • Interactive Reports
  • ALV Reports (ABAP List Viewer)

Classical Report Events

Classical reports follow a standard event sequence. The main business logic is typically written in START-OF-SELECTION.

EventPurpose
LOAD-OF-PROGRAMLoads the program into memory for execution.
INITIALIZATIONInitialize default values of program variables and selection-screen variables.
AT SELECTION-SCREEN OUTPUTModify the selection screen dynamically based on user action.
AT SELECTION-SCREENValidate multiple fields on the selection screen.
AT SELECTION-SCREEN ON <field>Validate a single input field on the selection screen.
AT SELECTION-SCREEN ON VALUE-REQUESTProvide search help (F4) for an input field.
AT SELECTION-SCREEN ON HELP-REQUESTProvide help information (F1) for an input field.
START-OF-SELECTIONMain event — write business logic and data fetching here.
END-OF-SELECTIONIndicates START-OF-SELECTION has ended.
TOP-OF-PAGEDisplay constant page heading on all pages.
END-OF-PAGEProvide constant footer information across pages.

💡 If you don’t code START-OF-SELECTION explicitly, ABAP assigns your “top-level” statements to it implicitly.

Interactive Reports

Interactive report programming allows users to drill down: a secondary list is displayed when the user interacts with the primary list.

  • AT LINE-SELECTION: triggered when user double-clicks an item in output
  • AT USER-COMMAND: handle custom user actions / toolbar changes
  • TOP-OF-PAGE DURING LINE-SELECTION: header for secondary list pages
  • AT PF<n>: function key press (F1–F12)
  • SET PF-STATUS: attach custom GUI status

📌 Tip: Use HIDE technique to store key fields from a selected line and retrieve details in AT LINE-SELECTION.

ALV Reports (ABAP List Viewer)

ALV is widely used for displaying SAP data with built-in features like sorting, filtering, totals, column reordering, and export.

Types

  • Simple ALV — normal grid/list display
  • Blocked ALV — multiple lists displayed consecutively
  • ALV Hierarchical Sequential — hierarchical parent/child display

Common Function Modules

Function ModuleUse
REUSE_ALV_GRID_DISPLAYDisplay results in a grid
REUSE_ALV_LIST_DISPLAYDisplay results in a list
REUSE_ALV_FIELDCATALOG_MERGEBuild/merge field catalog

Control Level Processing (AT Events)

Control level events are used within a LOOP on a (typically sorted) internal table to process grouped data.

EventTriggered When
AT FIRSTFirst row of internal table is read
AT NEW <field>Value of the specified field changes
AT END OF <field>End of group with same field value
AT LASTLast row of internal table is read

💡 Used for group headers, subtotals, and group footers (after sorting by the grouping field).

Interview Q&As

QWhat is a Classical Report in ABAP? How is it different from Interactive Report?
AClassical Report displays data in a flat list format with no drill-down. Interactive Report allows user interaction (like double-click) to generate secondary (detail) lists using events such as AT LINE-SELECTION and AT USER-COMMAND.
QWhat are the events in a Classical Report?
ALOAD-OF-PROGRAM, INITIALIZATION, AT SELECTION-SCREEN OUTPUT, AT SELECTION-SCREEN, AT SELECTION-SCREEN ON <field>, AT SELECTION-SCREEN ON VALUE-REQUEST, START-OF-SELECTION, END-OF-SELECTION, TOP-OF-PAGE, END-OF-PAGE.
QDifference between AT SELECTION-SCREEN and AT SELECTION-SCREEN ON <field>?
AAT SELECTION-SCREEN validates multiple fields together after input is collected. AT SELECTION-SCREEN ON <field> validates a specific field when the user leaves that field.
QWhat is an ALV Report and its types?
AALV (ABAP List Viewer) is a standard display framework with sorting, filtering, totals, export, etc. Types: Simple ALV, Blocked ALV, and Hierarchical Sequential ALV.
QWhat is SY-SUBRC? What value indicates success?
ASY-SUBRC is the return code for the last statement. 0 means success; non-zero typically means failure/not found.
QWhat are Variants in ABAP reports?
AVariants save selection-screen inputs (parameters/ranges) so users can reuse them—especially useful for background jobs.
