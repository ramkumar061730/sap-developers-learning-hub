SAP ABAP Basics

Foundational concepts every SAP developer must know — from what SAP is, to architecture, messages, and why ABAP exists.

1. What is SAP?

SAP stands for System Applications and Products in Data Processing. It is a German company founded in 1972 by five former IBM employees (Wellenreuther, Hopp, Hector, Plattner, and Tschira).

SAP is known as ERP software because it integrates all key business functions — finance, logistics, HR, and sales — into a single system, enabling real-time data sharing across departments.

Advantages of SAP

  • Eliminates duplicate data
  • Business processes are standardized
  • Real-time data access across all departments
  • Supports decision-making with accurate, integrated data
  • Scalability for large enterprises
  • Multi-language and multi-currency support

2. What is ABAP?

ABAP stands for Advanced Business Application Programming. It is a fourth-generation programming language (4GL) used for development and customization in the SAP software environment.

💡 ABAP is SAP's proprietary programming language, used to build custom applications on top of the SAP platform — reports, interfaces, forms, enhancements (RICEF/W).

RICEF/W — Primary ABAP Development Objects

  • Reports — Custom reports for business users
  • Interfaces — Data exchange between SAP and external systems
  • Conversions — Data migration from legacy systems
  • Enhancements — Extending standard SAP functionality
  • Forms — Output documents like invoices, POs
  • Workflow — Automated business process routing

3. Key SAP Components

Component Full Name Purpose
ECC SAP ERP Central Component Core ERP with FI, CO, MM, SD, PP, HR modules. Predecessor to S/4HANA.
GTS Global Trade Services Manages import/export trade processes in SAP ERP.
ERP Enterprise Resource Planning Manages day-to-day business: accounting, project management, risk, etc.
S/4HANA SAP S/4HANA Next-gen ERP built on HANA in-memory database.

4. SAP R/3 — 3-Tier Architecture

SAP R/3 means Real-time Data Processing (R = Real-time). It follows a 3-tier architecture:

Layer Role What Runs Here
Presentation Layer User Interaction SAP GUI, web browser, mobile device — any input device communicating with the end-user
Application Layer Business Logic ABAP programs are executed here; work processes (dialog, background, update, spool, enqueue) run here
Database Layer Data Persistence Stores all data: ERP data, program metadata, configuration; optimized for performance and security

💡 The 3-tier model separates concerns for scalability, performance, and security. Application and Database layers run on servers; only the Presentation layer is on the user's machine.

5. Types of SAP Messages

Code Type Description
A ABEND (Termination) Terminates the program immediately. Most severe.
I Information Displays information to the user; processing continues.
E Error Stops processing; user must correct the error before continuing.
S Status Displays a status message in the status bar; processing continues.
W Warning Alerts the user but allows processing to continue.
X Runtime Error Causes a short dump (runtime error / ST22).

6. Client Dependent vs Client Independent Data

Type Description Examples
Client Dependent Data specific to an individual client. Changes affect only that client. Business data, Customizing settings, Master data
Client Independent Shared across all clients in the system. Changes affect the entire SAP landscape. ABAP programs, Function Modules, Repository objects

7. Why is ABAP Development Needed?

Standard SAP covers ~80% of business needs. The remaining 20% requires custom development. ABAP allows developers to:

  • Create custom reports for end-users to easily access data
  • Build interfaces for data exchange with external systems
  • Migrate legacy data into SAP (conversions)
  • Enhance standard SAP without modifying core code (user exits, BADIs)
  • Generate output forms (invoices, delivery notes)
  • Automate business processes via Workflow

Interview Q&As

QWhat does SAP stand for and when was it founded?
ASAP stands for System Applications and Products in Data Processing. It was founded in 1972 by five former IBM employees (Wellenreuther, Hopp, Hector, Plattner, and Tschira) in Mannheim, Germany.
QWhat is ABAP and why is it used?
AABAP (Advanced Business Application Programming) is SAP's proprietary 4th generation programming language (4GL). It is used to develop custom programs, reports, interfaces, forms, and enhancements on the SAP platform. It allows developers to extend standard SAP functionality to meet specific business requirements.
QWhat are the three layers of SAP R/3 architecture?
A1) Presentation Layer — handles user interaction (GUI/browser/mobile). 2) Application Layer — contains business logic, processes ABAP programs; this is where work processes run. 3) Database Layer — stores all data persistently (ERP data, program metadata, configuration). The three-tier model separates concerns for scalability, performance, and security.
QWhat is the difference between client-dependent and client-independent data?
AClient-dependent data exists per client — changes in one client do not affect others (e.g., business data, customizing). Client-independent data is shared across all clients in the SAP system (e.g., ABAP programs, function modules, repository objects). Any change to client-independent data impacts all clients.
QWhat are the six types of SAP messages? Which is the most severe?
AA (Abend/Termination — most severe), I (Information), E (Error), S (Status), W (Warning), X (Runtime Error causing short dump). The most severe is type 'A' as it immediately terminates the program.
QWhat does RICEF/W stand for in SAP?
AReports (custom reports), Interfaces (data exchange between systems), Conversions (data migration from legacy systems), Enhancements (extending standard SAP), Forms (output documents like invoices), Workflow (automated business process routing). These are the primary development objects in SAP ABAP.
